입력양식실습

form 태그: 사용자의 입력을 받는 화면을 구성할 때 사용하는 태그로
입력한 데이터를 다른 페이지로 보낼 수(전송할 수) 있는 태그.
예) 로그인할때 아이디와 비밀번호를 입력하여 보내는 경우
회원가입할 때 가입에 필요한 여러 회원정보를 입력하여 보내는 경우 등..

input 태그

text 한 줄 텍스트를 입력하는 태그
password 입력 텍스트가 보이지 않는 비밀번호 태그
hidden 실제 소스에는 있지만 화면에서는 보이지 않는 히든 태그
radio 여러항목 중에 한 개만 선택할 때 사용하는 태그
checkbox 여러항목을 모두 선택할 수 있는 태그
file 첨부할 파일을 선택하는 태그
submit 폼 태그내의 입력데이터를 전송하는 버튼 태그

form 태그를 통해 입력 데이터를 내보내려면(전송하려면) 반드시 name속성이 있어야 한다

form 태그의 action 속성값이 없거나 action 을 지정하지 않으면 자기자신페이지를 연결

입력태그의 자동완성 기능: autocomplete 로 기본은 on인 상태
자동완성기능을 사용하지 않으려면 off로 지정한다

전송되는 입력값들을 파라미터라고 한다
전송하는 데이터가 여러개인 경우 각 파라미터는 주소창에 & 로 구분되어 있다
파라미터가 노출이 되는 형태, 노출이 되지 않는 형태가 있는데
노출되는 형태는 method: get 방식(기본), 노출되지 않는 형태는 method: post 방식

placeholder: 텍스트 입력태그에 무엇을 입력하는 태그인지 보여지게 하는 속성

required: 반드시 입력해야만 하는 필수입력항목을 입력하지 않으면 submit되지 않도록 처리하려면 프로그래밍적으로 해결할 수도 있지만 태그 자체의 required 속성을 이용할 수도 있다

readonly: 입력태그이지만 변경 불가하게 처리하고자 할때 지정하는 속성
disabled: 입력태그이지만 변경 불가하게 처리하고자 할때 지정하는 속성 차이점: readonly 속성을 갖는 태그는 submit 시 데이터가 전송되어지지만 disabled 속성을 갖는 태그는 submit 시 데이터가 전송되지 않는다

파일을 첨부하여 전송하는 경우는 반드시 지정해야 할 속성
1. method 를 post 로 지정해야만 한다.
2. enctype 을 multipart/form-data 을 지정해야만 한다
